# 테스트 시나리오 문서

이 문서는 홈페이지 관광지 목록 기능의 테스트 시나리오를 정리합니다.

## 테스트 환경

- **브라우저**: Chrome, Safari, Firefox (최신 버전)
- **디바이스**: 모바일, 태블릿, 데스크톱
- **테스트 모드**: 개발 환경 (검증 로직 활성화)

## 1. 필터 + 검색 + 페이지네이션 조합 테스트

### 1.1 지역 필터 + 검색 키워드 조합

**시나리오**: 서울 지역 선택 → "공원" 검색

**예상 결과**:
- 서울 지역의 "공원" 관련 관광지만 표시
- URL 파라미터: `areaCode=1&keyword=공원`
- 검색 결과 개수 정확히 표시
- 페이지네이션 동작 (결과가 15개 이상인 경우)

**검증 방법**:
1. 지역 필터에서 "서울" 선택
2. 검색창에 "공원" 입력 후 검색
3. URL 확인
4. 검색 결과 개수 확인
5. 페이지네이션 동작 확인

### 1.2 관광 타입 필터 + 검색 키워드 조합

**시나리오**: "관광지(12)" 선택 → "해변" 검색

**예상 결과**:
- 관광지 타입 중 "해변" 관련 항목만 표시
- URL 파라미터: `contentTypeId=12&keyword=해변`
- 다중 타입 선택 시 클라이언트 사이드 필터링 동작

**검증 방법**:
1. 관광 타입 필터에서 "관광지" 선택
2. 검색창에 "해변" 입력 후 검색
3. URL 확인
4. 결과 확인 (관광지 타입만 표시)

### 1.3 모든 필터 동시 적용 + 페이지네이션

**시나리오**: 서울 + 관광지/문화시설 + "박물관" 검색

**예상 결과**:
- 모든 필터 조건을 만족하는 항목만 표시
- 페이지 1에서 15개 항목 표시
- 페이지 2로 이동 시 다음 15개 항목 표시
- 총 개수 및 페이지 수 계산 정확성
- 각 페이지에서 필터 조건 유지

**검증 방법**:
1. 지역: 서울 선택
2. 관광 타입: 관광지, 문화시설 선택
3. 검색: "박물관" 입력
4. 페이지 1 확인 (15개 항목)
5. 페이지 2로 이동
6. 필터 조건 유지 확인

### 1.4 필터 변경 시 페이지 1로 리셋 확인

**시나리오**: 페이지 3에서 지역 필터 변경

**예상 결과**:
- URL에서 `page` 파라미터 제거 또는 `page=1`
- 목록이 첫 페이지 데이터로 업데이트
- 페이지네이션 UI가 1페이지로 리셋

**검증 방법**:
1. 페이지 3으로 이동
2. 지역 필터 변경
3. URL 확인 (`page` 파라미터 없음)
4. 페이지네이션 UI 확인 (1페이지 하이라이트)

### 1.5 검색 실행 시 페이지 1로 리셋 확인

**시나리오**: 페이지 5에서 새로운 키워드 검색

**예상 결과**:
- URL에서 `page` 파라미터 제거
- 검색 결과 첫 페이지 표시
- 기존 필터 조건 유지 (지역, 타입 등)

**검증 방법**:
1. 페이지 5로 이동
2. 새로운 키워드 검색
3. URL 확인 (`page` 파라미터 없음)
4. 필터 조건 유지 확인

## 2. 지도-리스트 연동 테스트

### 2.1 리스트 항목 클릭 → 지도 이동 및 마커 강조

**시나리오**: 데스크톱에서 리스트 항목 클릭

**예상 결과**:
- 지도가 해당 관광지 위치로 이동 (panTo)
- 해당 마커가 강조 표시 (다른 마커와 구분)
- 마커 인포윈도우 자동 표시
- 리스트에서 선택된 항목 하이라이트
- 여러 항목 연속 클릭 시 마지막 항목으로 이동

**검증 방법**:
1. 데스크톱 화면에서 리스트 항목 클릭
2. 지도 이동 확인
3. 마커 강조 확인
4. 인포윈도우 표시 확인
5. 리스트 항목 하이라이트 확인

### 2.2 마커 클릭 → 리스트 항목 강조

**시나리오**: 지도에서 마커 클릭

**예상 결과**:
- 인포윈도우 표시
- 리스트에서 해당 항목으로 스크롤 (필요시)
- 리스트 항목 하이라이트
- 여러 마커 연속 클릭 시 마지막 선택 항목 유지

**검증 방법**:
1. 지도에서 마커 클릭
2. 인포윈도우 표시 확인
3. 리스트 항목 하이라이트 확인

### 2.3 모바일에서 리스트 클릭 시 지도 탭으로 전환

**시나리오**: 모바일 화면 (< 1024px)에서 리스트 항목 클릭

**예상 결과**:
- 자동으로 "지도" 탭으로 전환
- 지도에 해당 관광지 표시
- 마커 강조 및 인포윈도우 표시
- "목록" 탭으로 돌아가도 선택 상태 유지

**검증 방법**:
1. 모바일 화면에서 리스트 항목 클릭
2. 지도 탭으로 전환 확인
3. 마커 표시 확인
4. 목록 탭으로 돌아가기
5. 선택 상태 유지 확인

### 2.4 선택된 관광지 하이라이트 유지

**시나리오**: 항목 선택 후 필터 변경

**예상 결과**:
- 필터 변경 시 선택 상태 초기화

**검증 방법**:
1. 항목 선택
2. 필터 변경
3. 선택 상태 초기화 확인

## 3. 페이지네이션 테스트

### 3.1 페이지 번호 클릭 시 올바른 데이터 표시

**시나리오**: 총 50개 항목, 페이지당 15개 (총 4페이지)

**예상 결과**:
- 페이지 1 클릭 → 1-15번 항목 표시
- 페이지 2 클릭 → 16-30번 항목 표시
- 페이지 3 클릭 → 31-45번 항목 표시
- 페이지 4 클릭 → 46-50번 항목 표시
- URL 파라미터 `page=2`, `page=3` 등 정확성
- 각 페이지에서 총 개수 표시 정확성 ("총 50개 중 16-30개 표시")

**검증 방법**:
1. 각 페이지 번호 클릭
2. 표시된 항목 범위 확인
3. URL 파라미터 확인
4. 총 개수 표시 확인

### 3.2 이전/다음 버튼 동작 확인

**시나리오**: 페이지 1에서 시작

**예상 결과**:
- "이전" 버튼 비활성화 (disabled)
- "다음" 버튼 클릭 → 페이지 2로 이동

**검증 방법**:
1. 페이지 1에서 "이전" 버튼 비활성화 확인
2. "다음" 버튼 클릭
3. 페이지 2로 이동 확인

### 3.3 첫 페이지/마지막 페이지 경계 처리

**시나리오**: 페이지 1에서 "이전" 버튼 클릭 시도

**예상 결과**:
- URL에 `page=0` 또는 음수 값이 들어가지 않음

**검증 방법**:
1. 페이지 1에서 "이전" 버튼 클릭 시도
2. URL 확인 (변경 없음)

### 3.4 URL 직접 입력 시 올바른 페이지 표시

**시나리오**: `/?page=2` 직접 입력

**예상 결과**:
- 페이지 2 데이터 표시
- 페이지네이션 UI에서 페이지 2 하이라이트

**검증 방법**:
1. URL에 `?page=2` 직접 입력
2. 페이지 2 데이터 표시 확인
3. 페이지네이션 UI 확인

### 3.5 브라우저 뒤로가기/앞으로가기 동작 확인

**시나리오**: 페이지 1 → 2 → 3 이동 후 뒤로가기

**예상 결과**:
- 페이지 2로 이동
- 페이지 2 데이터 표시
- URL이 `?page=2`로 업데이트

**검증 방법**:
1. 페이지 1 → 2 → 3 이동
2. 뒤로가기 버튼 클릭
3. 페이지 2로 이동 확인
4. 데이터 표시 확인

## 4. 검색 기능 테스트

### 4.1 검색 결과 개수 표시 정확성

**시나리오**: "공원" 검색 → API에서 반환된 totalCount 확인

**예상 결과**:
- 검색 결과 헤더에 "총 X개" 정확히 표시
- 페이지네이션의 총 개수와 일치

**검증 방법**:
1. "공원" 검색
2. 검색 결과 헤더 확인
3. 페이지네이션 총 개수 확인

### 4.2 검색 결과 없음 메시지 표시

**시나리오**: 존재하지 않는 키워드 검색 (예: "asdfghjkl")

**예상 결과**:
- "검색 결과가 없습니다" 메시지 표시
- "다른 키워드로 검색해보세요" 안내 문구
- 빈 상태 아이콘 (Search 아이콘) 표시
- 지도에 마커 없음
- 페이지네이션 숨김 처리

**검증 방법**:
1. "asdfghjkl" 검색
2. 빈 상태 메시지 확인
3. 지도 확인 (마커 없음)
4. 페이지네이션 확인 (숨김)

### 4.3 검색 + 필터 조합 결과 정확성

**시나리오**: "박물관" 검색 + 서울 지역 필터

**예상 결과**:
- 서울 지역의 "박물관" 관련 항목만 표시
- 다른 지역의 "박물관"은 제외
- URL 파라미터에 검색어와 필터 모두 포함

**검증 방법**:
1. 서울 지역 선택
2. "박물관" 검색
3. 결과 확인 (서울 지역만)
4. URL 확인

## 5. 필터 기능 테스트

### 5.1 지역 필터 변경 시 목록 업데이트

**시나리오**: 전체 → 서울 선택

**예상 결과**:
- 서울 지역 관광지만 표시
- URL에 `areaCode=1` 추가
- 지도 중심이 서울로 이동 (또는 관광지 범위로 조정)
- 마커가 서울 지역 관광지만 표시
- 페이지 1로 리셋

**검증 방법**:
1. 전체 상태에서 서울 선택
2. 목록 업데이트 확인
3. URL 확인
4. 지도 확인

### 5.2 관광 타입 다중 선택 동작

**시나리오**: 관광지(12) 선택 → 문화시설(14) 추가 선택

**예상 결과**:
- 관광지 + 문화시설 모두 표시
- URL에 `contentTypeId=12,14` 추가
- 관광지 선택 해제 시 문화시설만 표시
- 모든 타입 선택 해제 시 모든 타입 표시

**검증 방법**:
1. 관광지 선택
2. 문화시설 추가 선택
3. 결과 확인
4. URL 확인
5. 선택 해제 테스트

### 5.3 필터 초기화 기능

**시나리오**: 여러 필터 적용 후 "초기화" 버튼 클릭

**예상 결과**:
- 모든 필터 해제 (지역, 타입, 반려동물, 정렬)
- 검색어는 유지 (설계에 따라)
- URL이 기본 상태로 리셋 (`/` 또는 `/?keyword=xxx`)
- 페이지 1로 리셋
- 전체 관광지 목록 표시

**검증 방법**:
1. 여러 필터 적용
2. "초기화" 버튼 클릭
3. 필터 해제 확인
4. URL 확인

### 5.4 필터 상태 URL 파라미터 동기화

**시나리오**: 필터 변경 시

**예상 결과**:
- URL이 즉시 업데이트 (pushState, 리로드 없음)
- 브라우저 히스토리에 추가
- URL 복사 후 새 탭에서 열기 시 동일한 필터 적용

**검증 방법**:
1. 필터 변경
2. URL 즉시 업데이트 확인
3. 뒤로가기 동작 확인
4. URL 복사 후 새 탭에서 열기

## 테스트 결과 기록

각 테스트 시나리오를 실행한 후 다음 정보를 기록하세요:

- **테스트 날짜**: YYYY-MM-DD
- **테스트 환경**: 브라우저, 디바이스
- **결과**: ✅ 통과 / ❌ 실패
- **이슈**: 발견된 문제점 (있는 경우)
- **해결 방법**: 문제 해결 방법 (있는 경우)
